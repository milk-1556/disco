Questions

The honest answers.

Will building twice create duplicate channels? +

No. Builds are idempotent — they key off a manifest, so running the same build again reconciles to the same state instead of re-creating. If a build crashes mid-way, the next run resumes from the manifest rather than starting over.

What can't be cloned automatically? +

Anything Discord's API can't reproduce — third-party bot configurations, member data, boost perks, interactive panels, and Discovery. Disco never pretends to clone these. Each is surfaced as a manual step with a clear reason, so you always know exactly what's left to do by hand.

Can I see what will happen before I commit? +

Always. Every build supports a dry-run that writes nothing and produces the full report: what would be created, what would be skipped, and the manual steps. A pre-flight authority check also confirms the bot has every permission needed before any live run.

Do I need to give Disco my Discord token? +

You bring your own bot. You invite it to the source and target guilds and run the authority check from the dashboard. Disco drives the build through your bot — and you can rehearse the whole pipeline against an in-memory mock guild before ever pointing it at a real server.

How does the rebrand avoid breaking things? +

Find-and-replace is camelCase- and url-slug-aware, brand tokens are detected and pre-filled, and color and link maps are explicit. Crucially, every swap is previewed before a single channel is built — nothing changes silently.

What does the client actually receive? +

A polished, branded delivery page in their logo and colors — optionally password-gated and shareable as a link. It lists the scope delivered, gives one-click bot re-invite URLs, and walks them through transferring ownership of the server to themselves.
